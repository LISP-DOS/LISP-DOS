Resumo da Arquitetura

O LISP-DOS é um sistema operacional construído do zero (from scratch) projetado para emular a simplicidade de processo-único, porém embarcando uma arquitetura nativamente 64-bits (x86_64), paginação multinível, e um modelo robusto de segurança e isolamento usando anéis de privilégio (Ring 0 vs Ring 3).

Ao invés de depender das antigas interrupções de BIOS (INT 10h, INT 13h), nós controlamos o Hardware Diretamente através de uma camada moderna de abstração (HAL), gerenciando portas I/O, MMIO, e o barramento PCI/USB diretamente em Long Mode. O Kernel é desenhado para prevenir agressivamente Race Conditions e CPU Deadlocks por meio do uso calculado de salvaguardas de Assembly (como sti/hlt atômicos em loops de I/O) e proteção de estado contra perdas de contexto de User Space (preservando perfeitamente a System V ABI e registradores callee-saved).

Organização do Desenvolvimento (Árvore de Diretórios)

O repositório está subdividido rigorosamente para garantir a separação entre as camadas lógicas. Evite cruzar responsabilidades (ex: O FAT16 não deve invocar portas de VGA diretamente).

  • /src/hal/ Abstrações de hardware. Drivers de baixo nível (vga.c, ata.c, keyboard.c, pci.c, usb.c).
  • /src/kernel/ Cérebro do OS. Módulos centrais (main.c, syscall.c, process.c, fat16.c) e gerência de memória (pmm.c, vmm.c).
  • /src/tools/ Programas do espaço de usuário Ring-3 (editor.c, lisp.c, syscalls.h). Estes geram executáveis ELF autônomos.
  • /limine/ O Bootloader. Realiza a passagem do UEFI/BIOS para nosso Kernel Long Mode.
  • /doc/ & /dev-guide-html/ Documentações técnicas extensas (como este site).
  • Makefile Orquestrador de compilação. Cuida da geração dos artefatos (kernel.elf, hdd.img, msdos-lisp.iso).
