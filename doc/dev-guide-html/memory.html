Gerenciamento de Memória

Documentação Técnica do Kernel LISP-DOS

Visão Geral da Arquitetura

A alocação de memória no LISP-DOS é dividida em três pilares arquiteturais altamente desacoplados: O Gerenciador Físico (PMM), o Gerenciador Virtual (VMM) e a Alocação Dinâmica do Kernel (Heap/Kalloc). Diferente do real mode dos sistemas antigos que acessavam segmentos diretos de 16-bits, o LISP-DOS roda integralmente em modo longo (64-bits) e faz uso extensivo do HHDM (Higher Half Direct Map).

1. PMM (Physical Memory Manager)

Localizado em src/kernel/pmm.c. É a base granular que toma controle absoluto sobre os pentes de RAM reais instalados no computador, convertendo as seções relatadas como "Livres" pelo Bootloader (Limine) em um gigantesco Bitmap (mapa de bits).

Inicialização e Estrutura

A função pmm_init() itera o array de resposta do protocolo Limine. Toda página de memória mede exatos 4096 bytes (4 KB). O Bitmap armazena 1 para páginas alocadas/reservadas e 0 para páginas livres prontas para uso.

Funções Principais

  • void* pmm_alloc_page(): Procura sequencialmente o primeiro bit '0' no bitmap de memória, marca-o como '1' (ocupado) e retorna seu endereço de Memória Física base.
  • void* pmm_alloc_pages(size_t count): Procura por uma fileira contínua e ininterrupta de count páginas para buffers que requerem contiguidade a nível físico (muito utilizado em transfers de DMA que não entendem memória virtual).
  • void pmm_free_page(void* ptr): Desmarca matematicamente a página devolvendo a posse da RAM ao sistema. Operação assintótica em O(1).

2. VMM (Virtual Memory Manager)

Localizado em src/kernel/vmm.c. Opera exclusivamente as árvores de paginação de 4 níveis introduzidas pelos processadores x86_64 (PML4, PDPT, PD, PT).

O Mapeamento HHDM

Por questões de isolamento e simplicidade no tráfego Kernel-Ring, toda a Memória Física RAM existente é rigorosamente espelhada na Memória Virtual a partir de um offset fixo altíssimo. O Kernel acessa dados do User-Space injetando a constante do HHDM, ignorando assim a necessidade de usar os antigos seletores FS/GS da arquitetura x86.

Funções Principais Proteção de Anéis

Função Propósito
vmm_init()Inicia o PageMap raiz (Kernel). Identity map dos primeiros 4GB (para fallback) e carrega as bases MMIO do Hardware.
vmm_new_pagemap()Usada no modulo process.c. Instancia uma nova PML4 (diretório isolado) preenchendo as entradas Top-Half (Kernel) e limpando o Bottom-Half, conferindo total isolamento para um novo programa de modo usuário.
vmm_map_page()Amarra virtual-físico. Navega na árvore (criando sub-tabelas PDPT/PD/PT sob demanda usando pmm_alloc_page()) e injeta o flag PAGE_USER ou PAGE_WRITE.
vmm_switch_pagemap()Carrega as instruções assembly para gravar no registrador CR3 instruindo a MMU da CPU a aplicar o novo layout virtual. Invalida os TLBs.

Validadores de Segurança (Syscalls)

O VMM é munido de validadores preventivos em vmm_check_user_ptr e vmm_check_user_string. Elas percorrem toda a árvore do CR3 ativo validando se o ponteiro vindo de fora possui efetivamente a bitmask PAGE_USER e PAGE_PRESENT, evitando exploração de acessos maliciosos.

3. Kernel Heap (Alocador Dinâmico)

Localizado em src/kernel/heap.c. É o provedor central das famosas rotinas kmalloc e kfree (equivalentes ao padrão libc, mas reservadas somente ao Kernel Space).

Algoritmo de Lista Encadeada (Linked-List Block)

Durante a heap_init(), uma série contínua de páginas do PMM é requisitada e transformada em uma grande arena. O heap implementa blocos metrificados possuindo Cabeçalhos (Headers) escondidos, permitindo fatiamentos complexos.

struct heap_block {
    size_t size;           // Tamanho dos dados utilizáveis
    int free;              // Booleano indicando vacância
    struct heap_block* next; // Ponteiro para o nó vizinho
};

Dinâmica de Aglutinação (Merge)

Para evitar a tão danosa "Fragmentação do Heap" (onde pequenos buracos estraçalham as alocações), a função kfree checa matematicamente se o bloco next vizinho ou anterior também está livre. Caso esteja, funde (merges) as barreiras convertendo-os de volta em um grande bloco consolidado (Deflacting).
