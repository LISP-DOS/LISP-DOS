Módulo String & Memory

Documentação Técnica do Kernel LISP-DOS

Visão Geral

Independência (Freestanding): Como o Kernel LISP-DOS é compilado com as flags -fno-builtin e -ffreestanding, ele não possui acesso à biblioteca padrão do C (libc). O módulo string.c prove exatamente a base fundamental de manipulação de bytes que o Kernel precisa para inicializar estruturas, comparar nomes de arquivos no FAT16 e copiar buffers de memória.

A biblioteca implementa operações clássicas de ponteiros de modo linear e contínuo. As operações são intencionalmente minimalistas em linguagem C sem otimizações Assembly SIMD/AVX (para não poluir o Kernel e manter compatibilidade cruzada e arquitetural).

Funções de Manipulação de Memória

Ferramentas responsáveis por manipular bytes brutos e estruturas (Structs) sem distinção do conteúdo, tratando tudo como blocos hexadecimais inexpressivos (tipo uint8_t).

memset

Assinatura: void* memset(void *s, int c, size_t n)

Preenche uma área de memória específica com um byte constante.

  • s: Ponteiro base para o bloco de memória.
  • c: Valor do byte a ser repetido (internamente feito um cast para uint8_t).
  • n: Total de bytes consecutivos que serão substituídos.

Exemplo de uso: Essencial para zerar páginas HHDM (Higher Half Direct Map) recém alocadas via pmm_alloc_page() ou estruturas da Tabela de Páginas (PML4).

memcpy

Assinatura: void* memcpy(void *dest, const void *src, size_t n)

Copia n bytes absolutos de um endereço de origem para um endereço de destino.

  • Risco de Overlap: Diferente de funções avançadas como memmove, esta versão de cópia direta por laço while não possui defesa contra endereços que colidem (overlapping).

Exemplo de uso: Cópias rápidas de headers de Syscalls ou transposição de buffers e clusters decodificados do FAT16 para o espaço do usuário (Ring 3).

memcmp

Assinatura: int memcmp(const void *s1, const void *s2, size_t n)

Faz a validação aritmética de dois endereços de memória, comparando-os byte por byte.

Retorno: 0 se ambos os vetores forem precisamente iguais até o limite n. Caso encontre diferença, retorna a diferença algébrica do primeiro byte discrepante.

Funções de Manipulação de Strings

Focadas primariamente em arrays terminados em nulo (\0), comumente utilizadas para parser de caminhos de arquivos e argumentos.

strlen

Assinatura: size_t strlen(const char *s)

Avança ponteiro byte a byte até localizar um terminador ASCII Nulo (0x00), retornando a contagem escalar exata dos bytes transcorridos.

strcmp e strncmp

Assinaturas:
int strcmp(const char *s1, const char *s2)
int strncmp(const char *s1, const char *s2, size_t n)

Avaliam a equivalência entre cadeias de texto, mas com comportamentos baseados no terminador Nulo ao invés de contadores fixos (no caso do strcmp).

  • O strcmp interrompe imediatamente a verificação ao encostar no fim de alguma das strings.
  • O strncmp é a versão mais segura, pois impõe uma barreira máxima limitadora de iterações n, blindando o loop de invadir áreas protegidas do Kernel ou gerar vazamentos de memória (Buffer OOB).
