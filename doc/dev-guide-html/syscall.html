Módulo Syscall (System Calls)

Documentação Técnica do Kernel LISP-DOS

Visão Geral da Arquitetura

O módulo de Syscalls no LISP-DOS provê a interface segura entre o espaço de usuário (Ring 3) e o Kernel (Ring 0). A arquitetura foi projetada espelhando o padrão ABI do Linux x86_64, utilizando a instrução nativa syscall e sysretq de transição rápida.

Convenção de Chamada (ABI)

Quando um programa do usuário invoca uma rotina de sistema, os registradores devem ser configurados na seguinte ordem:

Registrador Propósito
RAXNúmero da Syscall (ID)
RDI1º Argumento
RSI2º Argumento
RDX3º Argumento
R104º Argumento
R85º Argumento
R96º Argumento
RCX & R11Endereço de Retorno (RIP) e RFLAGS

Preservação de Estado: Durante a transição, a instrução syscall destrói o conteúdo de RCX (salvando o RIP) e R11 (salvando o RFLAGS). No entanto, o wrapper em Assembly do Kernel (syscall_asm.S) atua proativamente protegendo o restante dos registradores. Ele empilha explicitamente os registradores temporários da ABI (como %r8 e %r9) e os demais parâmetros antes de transferir o contexto para o manipulador principal em C. Isso elimina o risco do Kernel corromper referências do espaço de usuário, como ponteiros alocados na .bss (evitando o infame Page Fault de violação de segurança do LISP).

Segurança e Isolamento (VMM Checker)

Proteção de Memória Rigorosa: O Kernel nunca confia em ponteiros passados pelo Ring 3. Todo endereço de memória fornecido sofre validação estrita antes da leitura/escrita.

As funções do Gerenciador de Memória Virtual (VMM) atuam como Gatekeepers em syscall.c:

  • vmm_check_user_ptr(vaddr, size, write_required): Garante que o buffer exista no PML4, não esteja no espaço do Kernel e possua as permissões corretas.
  • vmm_check_user_string(vaddr): Navega de forma segura pela paginação do usuário verificando o terminador NULL (\0) sem extrapolar limites.

Lista de Chamadas de Sistema

01 sys_write

Assinatura: void sys_write(const char *str)

Descrição: Imprime uma string terminada em nulo no terminal VGA.

02 sys_clear

Assinatura: void sys_clear(void)

Descrição: Limpa o buffer de vídeo do terminal VGA e reposiciona o cursor.

03 sys_set_cursor

Assinatura: void sys_set_cursor(int row, int col)

Descrição: Modifica as coordenadas de cursor através de hardware VGA ports.

04 sys_putchar

Assinatura: void sys_putchar(char c)

Descrição: Imprime e avança o offset. Lida com quebras de linha e rolagem de tela.

05 sys_set_color

Assinatura: void sys_set_color(uint32_t bg, uint32_t fg)

Descrição: Define paleta de background e foreground.

06 sys_get_max_rows

Assinatura: int sys_get_max_rows(void)

Descrição: Retorna o número de linhas disponíveis no terminal texto (padrão 25).

07 sys_keyboard_getchar

Assinatura: char sys_keyboard_getchar(void)

Descrição: Bloqueia a execução do usuário aguardando input via PS/2 IRQ1. O Kernel lida proativamente com a CPU usando a dupla sti / hlt e delays de hardware de 1 instrução para impedir race conditions. Como a instrução syscall desabilita IRQs globalmente ao entrar em Ring 0, este mecanismo é vital para impedir que a CPU congele em deadlocks infinitos esperando pelo teclado.

08 sys_fat16_read_file

Assinatura: int sys_fat16_read_file(const char* filename, uint8_t* buffer)

Descrição: Lê e copia clusters de arquivo de disco físico para user-space buffer com bounds-checking.

09 sys_fat16_write_file

Assinatura: int sys_fat16_write_file(const char* filename, const uint8_t* buffer, uint32_t size)

Descrição: Escreve novos arquivos no diretório root alocando blocos no FAT.

10 sys_fat16_get_file_size

Assinatura: int sys_fat16_get_file_size(const char* filename)

Descrição: Captura metadados para preparar ponteiros de memória em alocação dinâmica do usuário.

60 sys_exit

Assinatura: void sys_exit(void)

Descrição: Encerra subrotina Ring 3 e devolve a thread ao Shell do LISP-DOS.
