Arquitetura do Módulo de Vídeo

Guia de Desenvolvimento e Renderização HAL (Hardware Abstraction Layer)

Visão Geral

Diferente dos clones clássicos de 16-bits operando em modo real, o LISP-DOS em Long Mode (x86_64) não possui acesso ao Hardware VGA Modo Texto padrão através de ponteiros MMIO em 0xB8000. Operamos utilizando a passagem Linear via protocolo Limine.

A emulação do console visual acontece através do mapeamento de um Array de Pixels gigante, onde o Kernel calcula as dimensões por software e renderiza os bits decodificando glifos numéricos ASCII (rasterização bit-a-bit) utilizando a lib font8x8_basic.h e aplicando escalas de UI virtual.

size_t start_x = terminal_column * (8 * FONT_SCALE);
size_t start_y = terminal_row * (8 * FONT_SCALE);

fb_ptr[fb_y * (fb->pitch / 4) + fb_x] = color;

API de Cores VGA Virtuais

Conversão VGA (0-15) para TrueColor (RGB 24-bits)

Tradicionalmente as cores possuíam 4 bits (0 a 15). Como nossa placa de vídeo nativa está em modo moderno, a função vga_set_color faz o mapeamento transparente de um bitcode EGA para RGB hexadecimais.

0x0 - Preto
0x1 - Azul
0x2 - Verde
0x3 - Ciano
0x4 - Vermelho
0x7 - Cinza Claro
0xF - Branco
