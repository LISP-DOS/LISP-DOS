1. lisp.c - O Interpretador LISP

O LISP-DOS conta com um interpretador funcional que é ativado utilizando o comando lisp <arquivo> no shell.

Arquitetura do Interpretador:

  • AST (Árvore Sintática): Implementada sob os conceitos da linguagem LISP tradicional (S-expressions, átomos e listas). As estruturas em C fazem alocações dinâmicas de nós através de chamadas em heap.c para interpretar o parênteses do LISP, onde o primeiro elemento da lista sempre é o operador ou função a ser invocada.
  • Execução e Parsing: O sistema lê o buffer do arquivo em memória via fat16_read_file() ou lendo o Initramfs local e submete o texto ao lexer do LISP.
  • Suporte Atual: Permite definições de funções matemáticas globais e avaliações aninhadas como (defun soma ( x y ) ( + x y )) e as executa com ( soma 2 7 ) imprimindo o resultado.

2. editor.c - Editor de Texto Ncurses-like

Para manipular os scripts .li ou qualquer arquivo de texto sem precisar fechar o QEMU, foi criado um editor local ativado com o comando edit <arquivo>.

Ciclo de Vida:

  1. Lê o arquivo de texto se ele existir e aloca sua string no heap.
  2. Habilita o controle do framebuffer com uma barra de navegação no estilo Vi/Vim: -- NORMAL -- [i: Insert, w: Salvar, q: Sair, Setas: Navegar]
  3. No Modo Normal (botão Esc), o usuário usa atalhos ou setas do teclado PS/2 (lendo sequências de Escape no keyboard.c) para passear pela tela gráfica.
  4. Ao pressionar i, o usuário entra no Modo Insert, permitindo editar a string bruta do arquivo diretamente na memória.
  5. Ao pressionar w (Modo Normal), a string alterada é enviada ao fat16.c para ser persistida nos setores lógicos do hdd.img.
