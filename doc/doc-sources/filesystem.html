1. tarfs.c - Initramfs Local (Apenas Leitura)

O tarfs.c implementa um interpretador básico do formato de arquivo USTAR (Standard TAR). O bootloader Limine é instruído no arquivo limine.conf a carregar arquivos arbitrários como "módulos" no espaço de memória antes de executar o kernel.

O LISP-DOS utiliza isso como uma unidade de disco somente leitura (Initramfs). Ao iniciar, o código mapeia a localização desse arquivo TAR na RAM e lê os cabeçalhos de bloco de 512 bytes procurando nomes de arquivos (como utilitários ou o LEIAME.txt).

Funções chaves:

  • tarfs_init(): Procura o módulo TAR na memória do Limine.
  • tarfs_dir(): Lê sequencialmente o TAR mostrando o nome dos arquivos no terminal.
  • tarfs_type(): Encontra e imprime o conteúdo de um arquivo na tela.

2. fat16.c - Disco Rígido Persistente (Leitura/Escrita)

O arquivo fat16.c lida com sistemas de arquivos Microsoft FAT16. Diferente do TarFS, este componente possui permissão para leitura e escrita, garantindo que o que foi feito no computador não seja perdido no desligamento.

Ele opera em cima da HAL comunicando-se com a função ata_read_sector e ata_write_sector em ata.c.

A Arquitetura Lógica:

  1. Boot Sector (MBR/VBR): Lê o setor 0 para encontrar os parâmetros do disco (Tamanho do Cluster, Bytes por Setor, Reserva).
  2. FAT (File Allocation Table): Lê a tabela que indica onde o arquivo continua (fragmentação) no disco físico.
  3. Diretório Raiz (Root Directory): Percorre as entradas de 32 bytes procurando o nome em formato "8.3" (8 letras para o nome, 3 para a extensão, como SOMA LI ).

A persistência no LISP-DOS é garantida por funções dentro deste arquivo como fat16_write_file, que alocam clusters vazios na tabela FAT, gravam os dados nas regiões de dados do disco físico e criam a entrada do arquivo no diretório raiz.
