1. CPU e Interrupções (gdt.c, idt.c, pic.c, interrupts.S)

O funcionamento correto do kernel x86_64 depende de algumas tabelas da CPU:

  • GDT (Global Descriptor Table): Em gdt.c, o sistema configura a memória no modelo "flat", permitindo acesso total do kernel aos endereços de memória sem segmentação limitante.
  • IDT (Interrupt Descriptor Table): Definida em idt.c, mapeia os vetores de erro (exceções, como Divisão por Zero) e requisições de hardware para rotinas de software.
  • PIC (Programmable Interrupt Controller): O arquivo pic.c reprograma as interrupções base do hardware (IRQs). O timer (IRQ0) e o teclado (IRQ1) são mapeados em vetores na IDT que não colidem com as exceções de CPU (geralmente vetores 32+).
  • interrupts.S: Contém stubs em código Assembly para salvar os registradores da CPU de forma segura, chamar a rotina em C (ex: o driver de teclado) e restaurar o contexto logo após.

2. Dispositivos de Entrada e Saída (I/O)

Para ler e escrever dados nas portas de I/O do x86 (teclado e discos IDE), o sistema utiliza funções wrapper definidas em io.h que executam instruções assembly inb, inw, outb, outw.

3. vga.c e font8x8_basic.h (Driver de Vídeo)

O arquivo vga.c abstrai o acesso à tela, oferecendo funções confortáveis como vga_print e vga_putchar.

Ele trata o texto do console de duas formas (dependendo do que o Limine oferece): se for um framebuffer gráfico, o arquivo font8x8_basic.h contém um mapa de bits de 8x8 pixels para cada caractere ASCII, permitindo que a função "desenhe" as letras píxel por píxel na tela gráfica.

4. keyboard.c (Driver de Teclado PS/2)

Conectado ao canal IRQ1, esse driver recebe um byte da porta 0x60 toda vez que uma tecla é pressionada ou solta. O keyboard.c decodifica o 'scancode' recebido em um caractere ASCII correspondente, gerenciando modificadores como Shift e CapsLock.

O kernel consome essas letras usando keyboard_getchar(), que bloqueia o loop principal até que uma tecla esteja disponível no buffer.

5. ata.c (Driver de Disco IDE)

Este arquivo fornece leitura e escrita bruta em LBA (Logical Block Addressing) nos discos conectados às portas PATA/SATA do computador, usando o modo PIO legado através das portas de I/O 0x1F0-0x1F7. Sem isso, os sistemas de arquivos não teriam meio de persistir dados.
