1. main.c - O Ponto de Entrada

O arquivo main.c é responsável por iniciar o kernel após o carregamento pelo Limine. A função _start() coordena a inicialização do hardware, memória e os sistemas de arquivos.

Principais Funcionalidades:

  • Inicialização de Vídeo (vga_init).
  • Configuração da GDT e IDT (gdt_init, idt_init) para segmentação e interrupções.
  • Inicialização de memória física e dinâmica (pmm_init, heap_init).

    • O Loader é encarregado de parsear os cabeçalhos Elf64_Ehdr e Elf64_Phdr do executável. Ao contrário do DOS clássico que apenas alocava um segmento COM/EXE bruto, o LISP-DOS realiza uma inspeção forte: valida assinaturas (Magic Numbers), calcula matematicamente as estruturas para prevenir Buffer Over-Reads se o ELF mentir seus offsets, e nega categoricamente a inicialização se o binário apontar o seu e_entry para o Kernel Space. Só depois dessa validação ele mapeia os segmentos na memória virtual com as flags PAGE_USER | PAGE_WRITE.

  • Inicialização de drivers de disco (tarfs_init, fat16_init).
  • O Loop do Shell: Um loop infinito onde o kernel exibe o prompt C:\>, aguarda entradas do keyboard_getchar() e despacha os comandos (como echo, cls, dir, edit e lisp).

2. pmm.c - Gerenciador de Memória Física

O PMM (Physical Memory Manager) lê o mapa de memória fornecido pelo bootloader (através das tags do Limine) para entender quais endereços físicos de RAM estão disponíveis na máquina.

Funcionamento:

O pmm divide a memória usável em blocos de 4KB (páginas). Para rastrear essas páginas, ele normalmente utiliza um mapa de bits (bitmap) ou uma lista encadeada, garantindo que alocações do kernel não sobrescrevam áreas vitais, como o próprio código do kernel, framebuffers ou módulos carregados (initramfs).

3. heap.c - Gerenciador de Memória Dinâmica

O LISP-DOS não conta com as bibliotecas do C padrão (`stdlib.h`). Logo, as funções malloc e free são implementadas manualmente no heap.c.

Funcionamento:

Ele recebe uma grande região de memória inicializada pelo PMM e gerencia pedaços menores solicitados pelas aplicações (como o editor de textos ou os nós/listas do interpretador LISP). Cada bloco no heap contém um cabeçalho escondido indicando seu tamanho e se está livre ou ocupado. O comando free junta blocos livres adjacentes para evitar fragmentação.
