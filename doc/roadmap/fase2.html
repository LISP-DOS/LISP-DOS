Fase 2: Expansão de Hardware Moderno

Nesta fase o LISP-DOS se liberta das amarras dos computadores primitivos dos anos 80. O foco não é criar novas funcionalidades de usuário no LISP, mas sim escrever drivers complexos para interagir diretamente com o silício moderno usando o barramento PCIe.

Parte 2.1: Tempo e Relógios (RTC/PIT/APIC)

Problema Atual: O Kernel não possui noção da passagem de tempo. Processos não podem esperar sem travar (ocupando ciclos inúteis de CPU).

Tarefas:

  • Implementar o driver RTC (Real Time Clock) lendo as portas 0x70 e 0x71 (CMOS) para capturar o ano, mês, dia, horas e segundos reais.
  • Configurar o PIT (Programmable Interval Timer) na porta 0x43 para disparar a Interrupção IRQ0 em uma frequência determinística (ex: 1000 Hz = 1 milissegundo).
  • Como alternativa em placas modernas, inicializar e configurar o Local APIC Timer.
  • Adicionar a Syscall sys_sleep(millis) que utiliza hlt atômico enquanto aguarda os ticks avançarem no Kernel, poupando energia elétrica.
  • Adicionar a Syscall sys_get_time para aplicações Ring 3.

Parte 2.2: O Barramento PCIe e USB 3.0 (xHCI)

Problema Atual: Dependemos do emulador PS/2 da placa-mãe para o teclado. Periféricos puros USB não são reconhecidos.

Tarefas:

  • Criar um enumerador de barramento PCI / PCIe (Configuration Space Access) capaz de listar todos os hardwares presentes no PC por Vendor ID e Class Code.
  • Identificar a controladora xHCI (eXtensible Host Controller Interface), mapeando o endereço base da MMIO a partir do registrador BAR0 do PCIe.
  • Programar o reset da controladora xHCI, configurar o Device Context Base Address Array (DCBAA) e o anel de comandos (Command Ring).
  • Ativar a abstração das portas físicas da placa mãe, identificando se um teclado está plugado.
  • Escrever o mini-driver HID para extrair o report de *Keystrokes* vindos dos pacotes Transfer Ring do xHCI e traduzi-los para o buffer padrão ASCII do LISP-DOS, substituindo a porta 0x60 original.

Parte 2.3: Discos Ultra-Rápidos via SATA AHCI

Problema Atual: O arquivo fat16.c invoca o driver ata.c que opera em LBA28 usando I/O Blockings em portas. Esse é o modo mais lento possível (desenvolvido na década de 90).

Tarefas:

  • Através do scan do barramento PCIe criado na Parte 2.2, encontrar o controlador com a Classe de Armazenamento Serial ATA (Class Code 01h, Subclass 06h).
  • Alocar as tabelas de memória PRDT (Physical Region Descriptor Table).
  • Configurar a Command List e enviar o comando READ FPDMA QUEUED via portas AHCI para o disco rígido.
  • O hardware de disco escreverá o conteúdo diretamente na memória física através de DMA (Direct Memory Access), enviando um IRQ apenas quando finalizar. Isso eleva a banda de cópia para a casa dos Gigabytes por segundo.
  • Atualizar a rotina base das abstrações de HD em fat16.c para chamar internamente ahci_read_sector ao invés de ata_read_sector.
