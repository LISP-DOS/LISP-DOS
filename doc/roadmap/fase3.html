Fase 3: Maturidade de Sistema de Arquivos

A Fase 3 abandona o uso de nomes de arquivos brutos fixos (ex: "apenas o diretório raiz"). O objetivo é abstrair o armazenamento para parecer com um sistema operacional maduro onde aplicações lidam apenas com ponteiros genéricos e descritores (File Descriptors).

Parte 3.1: O Virtual File System (VFS)

Problema Atual: Para ler um arquivo do HD, o usuário chama a syscall `sys_fat16_read_file` amarrando toda a lógica à implementação de baixo nível daquele formato exato.

Tarefas:

  • Criar a camada VFS (Virtual File System) no Kernel com uma estrutura de pontos de montagem (`mount points`), ex: `/dev/sda` ou `/hd0`.
  • Introduzir o conceito de File Descriptors (FD) inteiros mantidos na struct do processo (Ring 3). Padrão UNIX: FD 0 = STDIN, FD 1 = STDOUT, FD 2 = STDERR. FDs maiores que 2 são arquivos abertos.
  • Adicionar novas Syscalls: sys_open(path, flags), sys_read(fd, buffer, size), sys_write(fd, buffer, size), sys_close(fd), sys_lseek(fd, offset, origin).
  • Com isso, LISP scripts e o Editor trabalharão com streams (fluxos) de dados gigantes sem precisarem alocar a memória total de um arquivo na RAM antes de começar a processá-lo.

Parte 3.2: Subdiretórios e Hierarquia no FAT16

Problema Atual: O FAT16 lê obrigatoriamente a partir de um limite fixo do "Root Directory Region".

Tarefas:

  • Modificar o driver fat16.c para interpretar atributos em diretórios (`0x10`).
  • Escrever a lógica recursiva para que caminhos em barra invertida (C:\DOCS\TEXTOS\ARQUIVO.TXT) ou normais (/docs/textos/arquivo.txt) resolvam dinamicamente buscando cluster a cluster através de subpastas na cadeia FAT.
  • Adicionar comandos nativos no shell main.c: cd <dir> para trocar de diretório corrente e mkdir <dir> para criar diretórios.

Parte 3.3: Suporte ao FAT32

Problema Atual: O FAT16 tem um limite teórico estrutural ao alocar grandes volumes (clusters se esgotam em drives modernos formatados em Windows).

Tarefas:

  • Criar o arquivo driver fat32.c para decifrar a FAT (File Allocation Table) em modo estendido com 32-bits de endereço por cluster.
  • Lidar com o FAT32 BPB (BIOS Parameter Block) estendido.
  • Registrar o fat32 no VFS (da Parte 3.1) e fazê-lo autodetectar a formatação analisando o tamanho da contagem de setores na inicialização.
