Fase 5: Multitarefa (Multitasking Preemptiva) e Rede

A Fase 5 é o estágio de iluminação. Aqui deixamos formalmente o paradigma MS-DOS estrito para trás, e passamos a agir como um kernel monolítico corporativo no estilo UNIX-Like moderno, permitindo processos simultâneos sem que um bloqueie a execução do outro.

Parte 5.1: Context Switching (Estruturas base do Scheduler)

Problema Atual: Ao chamar o LISP, o kernel joga a thread no lixo e entrega todo o controle da CPU para ele, morrendo de sede esperando ele invocar `sys_exit` de volta. O loop do shell é interrompido indefinidamente.

Tarefas:

  • Criar a lista encadeada (Queue) de Process Control Blocks (PCB) internamente em process.c. Cada bloco deve guardar todo o estado da CPU no momento da "Pausa" (Registradores 64-bits: RAX, RBX, RSI, RCX, R8..R15).
  • Armazenar individualmente a ID do VMM (`PML4` CR3 register) e o status da tarefa (Pronto, Bloqueado, Rodando, Zumbi).

Parte 5.2: O Motor do Escalonador (Timer Interrupts)

Problema Atual: Precisamos de uma "facada" assíncrona que arranque o controle das mãos dos programas de forma determinística repetidamente ao longo de milissegundos.

Tarefas:

  • No handler do PIT / Local APIC Timer (IRQ0 configurado na Fase 2), forçar a CPU a pular para o salvamento de contexto. O Kernel salvará os registradores do processo em execução no PCB no instante exato.
  • Adotar o algoritmo *Round Robin* para escolher qual será o próximo PCB da lista e injetar seus registradores no CPU (pop dinâmico e iretq) saltando o RIP da CPU perfeitamente de volta pro ponto salvo do próximo programa.
  • Conseguir rodar o Editor de Texto na tela enquanto em Background um código LISP processa dados pesados, dividindo milissegundos transparentemente do tempo da CPU para os dois rodarem.

Parte 5.3: Pilha de Rede (The Holy Grail)

Problema Atual: LISP-DOS é completamente isolado, sem noção do mundo ao redor.

Tarefas:

  • Usar o Bus PCI (Fase 2) para inicializar a placa de rede emulada nativa padrão do mercado, como a `Intel E1000` (PRO/1000) ou `Realtek RTL8139`.
  • Implementar a abstração base das camadas OSI de Enlace (Ethernet Frames com enderaçamento físico MAC).
  • Adicionar suporte de resposta a pacotes IPv4 com os cabeçalhos IP base (calcular UDP Checksums e enviar respostas ICMP/Ping).
  • Fornecer Syscalls nativas na Camada de Rede no interpretador LISP para instanciar Sockets (permitindo scripts acessarem o mundo real!).
