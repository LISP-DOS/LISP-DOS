Parte 6.1: Bootloader e Toolchain AArch64

1. Ambiente de Compilação (Cross-Compiler)

Como estamos no Windows (via WSL), os compiladores padrões geram código x86_64. Precisaremos instalar a Toolchain oficial da GNU para processadores ARM 64-bits (AArch64).

O pacote necessário no nosso Makefile será alterado de gcc para aarch64-none-elf-gcc ou aarch64-linux-gnu-gcc.

Em vez de gerar um arquivo .iso emulado com o Limine, nosso compilador gerará um binário planificado chamado kernel8.img, que é o nome de arquivo nativo que a placa Raspberry Pi 4 busca no SD Card por padrão para processadores de 64-bits.

2. A Anatomia do Boot do Raspberry Pi

No PC Clássico, o processador x86 acorda e lê a BIOS. No Raspberry Pi, o processador ARM fica desligado no início. Quem liga o sistema é a placa de vídeo (GPU VideoCore VI).

  • A GPU é ligada e executa seu Boot ROM (primeiro estágio).
  • Ela lê o Cartão SD formatado em FAT32.
  • Encontra e lê os binários fechados da Broadcom (bootcode.bin e start4.elf).
  • Lê o arquivo de texto config.txt.
  • Por fim, a GPU carrega o nosso código (kernel8.img) diretamente na RAM, liga a energia do processador ARM, e aponta o ponteiro de instrução da CPU (PC) para o endereço base da memória.

3. O Arquivo Linker (link.ld)

Por padrão da arquitetura Broadcom, o arquivo kernel8.img é jogado na RAM a partir do endereço de memória 0x80000. Diferente do kernel atual (carregado em 0xFFFFFFFF80000000), precisamos fornecer um Linker Script rígido garantindo que o cabeçalho do assembly puro seja o primeiríssimo comando na memória.

SECTIONS
{
    . = 0x80000;
    .text : { KEEP(*(.text.boot)) *(.text .text.*) }
    .rodata : { *(.rodata .rodata.*) }
    .data : { *(.data .data.*) }
    .bss : {
        . = ALIGN(16);
        __bss_start = .;
        *(.bss .bss.*)
        __bss_end = .;
    }
}

4. Inicialização em Assembly (.text.boot)

O Raspberry Pi liga a CPU com os 4 núcleos (Cores) ativados rodando o mesmo código simultaneamente. Para um OS inicial single-thread, devemos ler o ID do núcleo no registrador mpidr_el1, estacionar os núcleos 1, 2 e 3 em um loop infinito, e prosseguir com o carregamento da Stack apenas no núcleo 0.
