Parte 6.2: Exception Levels (EL) e Debug via UART

1. Descida de Privilégios (Exception Levels)

O ARM não utiliza os mesmos conceitos de "Ring 0" e "Ring 3" que o x86. Ele utiliza Exception Levels (EL) invertidos, onde o número maior tem mais poder.

LevelDescriçãoMapeamento no LISP-DOS
EL3Monitor Seguro (TrustZone)Ignorado pelo OS Base
EL2HypervisorO Raspberry Pi joga nosso código aqui na largada.
EL1Sistema Operacional (Kernel)Nós desceremos para este modo para rodar o Core.
EL0Aplicações (User Mode)Onde o LISP e o Editor vão rodar isolados.

A primeira missão estrutural no arquivo de assembly do kernel será manipular os registradores especiais spsr_el2 e elr_el2 e chamar a instrução eret (Exception Return) para forçar o processador a cair com segurança do nível de Hypervisor (EL2) para o nível de Kernel real (EL1), que nos garantirá as permissões padrão para escrever na tabela de memória.

2. UART (Universal Asynchronous Receiver-Transmitter)

No PC, podemos apenas escrever em 0xB8000 para mostrar letras na tela. Na placa Raspberry Pi física, configurar o HDMI dá muito trabalho nas primeiras semanas. A forma canônica de "falar" com a placa na Fase Zero é via os Pinos GPIO usando comunicação Serial (UART).

O Raspberry Pi 4 possui o hardware Mini UART compatível com o protocolo antigo 16550. É a forma mais robusta de exibir logs.

Procedimento do Driver Serial (hal/rpi_uart.c):

  1. No arquivo config.txt do cartão SD, adicionaremos a flag enable_uart=1, que pede para a GPU manter a frequência do Core Clock estável, necessária para o Mini UART funcionar sem gerar caracteres lixo.
  2. No código do LISP-DOS, acessaremos os registradores AUX_MU_IER_REG e AUX_MU_LCR_REG nas portas MMIO usando offsets mapeados.
  3. Vamos configurar o Baud Rate matematicamente (ex: 115200) baseado no Clock rate da GPU (típicamente 500 MHz no RPi 4).
  4. Programaremos os Pinos GPIO 14 (TX) e 15 (RX) para sua "Função Alternativa 5" para delegá-los ao Módulo UART ao invés da operação digital clássica de liga/desliga.

Isso substituirá a nossa função de teste vga_print provisoriamente por uart_print para atestar que o Kernel bootou e não travou.
