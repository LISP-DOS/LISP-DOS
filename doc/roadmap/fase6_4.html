Parte 6.4: Integração do LISP Core e Mailbox Framebuffer

1. Reativando a Interface Gráfica (GPU Mailbox)

No x86, o bootloader Limine fornecia a tela de pixels pra gente de forma mágica através de uma estrutura C pronta. No Raspberry Pi, o Kernel precisa "conversar" eletronicamente com a GPU VideoCore para negociar um monitor via conexão HDMI.

A arquitetura ARM da Broadcom utiliza um canal de mensagens chamado Property Mailbox (Canal 8). A implementação ocorrerá assim:

  • O OS do LISP alocará um array na memória com formato rígido de Tags (A TAG 0x00040003 requisita tamanho físico do display, a TAG 0x00040005 negocia a profundidade em 32-bits de cor, e a 0x00040001 pede a alocação do Framebuffer propriamente dito).
  • O CPU empurra o endereço de memória dessa mensagem estruturada para os registradores Mailbox MMIO de hardware (0xFE00B880).
  • A GPU responde injetando os ponteiros virtuais contendo o endereço alocado da tela no hardware nativo, trazendo as funcionalidades completas que perdemos no port.

Feito isso, nosso driver anterior nativo vga.c (que rasteriza textos e retângulos pixel por pixel) voltará a ligar milagrosamente, exigindo apenas que os ponteiros base sejam ajustados na struct inicial.

2. Portando Aplicações (Cross-Compiling Apps)

O Código fonte isolado das nossas ferramentas (lisp.c e editor.c) é 100% agnóstico a processador pois é puro "C". O trabalho agora é puramente de reestruturação de Build (Makefile).

  • Elas serão recompiladas usando a flag nativa do GCC ARM para gerar binários no formato puro ELF nativos para arm64.
  • O Interrupt Handler de exceptions de Kernel (atual syscall_asm.S x86) será reescrito na sintaxe ARM svc #0, que é a interrupção do tipo Supervisor Call usada pelo ARM para delegar transição Ring 3 (EL0) -> Ring 0 (EL1).

Conclusão da Fase 6

O término desta epopeia garantirá a você plugar um cartão SD formatado com a nossa imagem "kernel8.img" em um Raspberry Pi 4 físico de bancada, ligar o HDMI na televisão e ver a belíssima e icônica tela preta do LISP-DOS aguardando comandos do seu código nativo (LISP) sob hardware bare-metal brutal sem depender de virtualizações complexas.
